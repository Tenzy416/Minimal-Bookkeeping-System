## 一、目前做出了什麼？

> 具體描述目前可以運作的功能，不需要說「正在開發」，只填**已經能用**的部分。

| 功能 | 說明（一句話描述能做什麼） |
| ---- | -------------------------- |
| **收支紀錄新增 (F-01)** | 使用者可輸入金額、選擇收支類型、日期及備註，並儲存至 SQLite 資料庫。 |
| **歷史紀錄列表 (F-02)** | 自動由新到舊依日期排序，列出所有收支明細，支援備註換行及空狀態提示。 |
| **統計結餘總額 (F-03)** | 自動計算所有記帳紀錄，在畫面上方即時更新總收入、總支出與目前結餘。 |
| **前端輸入防禦校驗** | 自動阻擋負數、0 或非數字的無效金額輸入，並跳出紅色警告 (Toast)。 |
| **響應式佈局 (RWD)** | 針對平板與手機進行版面調校，手機版統計卡片自動轉為垂直堆疊且易於閱讀。 |

## 二、遇到哪些 Bug？

> Bug 是指「你預期它會這樣，但它做了別的事或是發生錯誤」。

| Bug 描述（發生了什麼？） | 目前狀態                |
| ------------------------ | ----------------------- |
| **[B-01] 歷史明細長文字溢出**：備註輸入過長時無法折行，會把右側金額擠出卡片。 | ✅ 已修好 |
| **[B-02] & [B-03] 行動版畫面擠壓**：手機上三個統計卡片太窄，導致文字與圖標重疊。 | ✅ 已修好 |
| **[B-04] 金額欄位缺乏防禦**：能寫入 `0` 或 `負數`，導致統計數據出現負值或異常。 | ✅ 已修好 |
| **[B-05] Flask 啟動路徑錯誤**：在非專案目錄下啟動 `app.py` 會導致 404 找不到 index.html。 | ✅ 已修好 |

## 三、遇到哪些困難？

> 可以是技術上不會、找不到資料、組員時間喬不攏、工具用不順⋯⋯什麼都可以寫。

| 困難描述 | 已嘗試什麼做法？ | 目前需要協助嗎？ |
| -------- | ---------------- | ---------------- |
| **1. 對 Flask 框架與資料庫操作不熟練** | 提早一週研讀官方文件，先從小型的 API 測試程式開始，與組員共同除錯。 | 否 |
| **2. 前後端傳遞 (JSON) 格式不合** | 在實作前，組員先開會訂好前後端 API 規格（欄位名稱與型態），落實規格化開發。 | 否 |
| **3. Git 代碼合併時產生衝突 (Conflict)** | 每次 Push 前先 Pull，並將前後端檔案確實拆分，降低多人同時編輯同檔案的機率。 | 否 |
